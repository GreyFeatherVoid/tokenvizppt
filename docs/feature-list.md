# tokenvizPPT 当前功能列表

更新时间：2026-05-16

本文档用于记录项目当前已经具备的产品能力、管理能力和运维能力。它偏向功能盘点，后续可以继续拆成用户手册、管理员手册和开发计划。

## 1. 产品定位

tokenvizPPT 是一个 AI PPT 生成、编辑和导出工具，支持用户通过主题、需求说明和上传文件生成可编辑的演示文稿。前端负责交互和展示，模型调用、API Key、文件解析、素材存储、后台任务和 PPTX 导出都放在 Python 后端。

## 2. 首页与工作台

- 首页已具备产品介绍、免费体验说明、注册/登录入口和“直接使用”入口。
- 首页可以展示管理员发布的公告。
- 未登录用户可以进入工作台体验生成能力。
- 登录用户可以进入“我的”查看账户、积分、邀请码、项目列表和积分流水。
- 管理员登录后可以看到管理入口。
- 工作台包含生成表单、进度面板、等待小游戏、历史项目和生成后的 PPT 编辑区。

## 3. PPT 生成

- 支持按主题和需求说明生成 PPT。
- 支持设置页数，当前前端限制为 1 到 40 页。
- 支持选择内置风格，并允许编辑本次生成使用的风格提示词。
- 支持自动选择输出语言。
- 生成任务由 Celery Worker 异步执行，前端通过状态接口和 SSE 事件展示进度。
- 生成任务会显示排队中、生成中、已完成等状态。
- 后端有 SlideSpec 生成、HTML 生成、版式校验、重试和异常记录能力。
- 支持并发生成多页，具体并发数由后端配置控制。
- 支持整套生成任务超时控制，超时任务会失败并释放用户的运行中额度。
- 失败任务会记录失败分类，便于管理员判断是超时、积分、模型配置、模型请求、版式校验、素材放置、生图还是系统异常。

每个风格包含中英文说明和生成提示词，前端会按当前界面语言获取对应风格信息。

## 5. 文件与素材

- 生成时支持上传 `txt`、`md`、`csv`、`pdf`、`docx`、`xlsx` 和图片文件作为上下文。
- 图片文件支持备注，并可以标记为“必须出现在 PPT 中”。
- 非图片文件不会作为 fallback image 插入。
- 生成后的项目支持继续上传素材。
- 素材可以插入当前幻灯片，也可以按用户说明由 AI 辅助放置。
- 后端负责文件存储、文件列表、素材更新和文件下载。

## 6. 图片分析与 AI 生图

- 上传图片可以由视觉模型分析，作为生成上下文。
- 用户可以勾选允许 AI 生成配图。
- 后端只有在配置开启、用户勾选、且具体页面确实需要图片时才会生成图片。
- AI 生图数量有单个 PPT 上限，由后端配置控制。
- 生图费用按积分规则扣除；当前生成任务整体成功后再扣除，失败或管理员取消不会扣除。

## 7. PPT 编辑

- 生成完成后可以预览整套 PPT。
- 支持选择单页幻灯片并查看页面元素。
- 支持对单页进行 AI 修改。
- 支持直接修改可编辑文本元素。
- 支持删除文本内容并保存为空文本。
- 支持插入图片素材。
- 支持让 AI 根据指令放置图片素材。
- 支持查看单页历史版本。
- 支持回滚到历史版本。

## 8. PPT 导出

- 支持导出可编辑 `.pptx` 文件。
- 导出任务由后台 Worker 异步执行。
- 前端会轮询导出任务状态，完成后提供下载链接。
- 导出的文件存储在后端 `storage` 目录下。

## 9. 历史项目

- 前端有最近项目列表。
- 支持打开历史项目。
- 支持删除历史项目。
- 未登录状态下目前仍有本地历史能力；后续如果要做公开首页案例区，建议把未登录历史与案例展示分开。
- 登录用户的项目会关联到账户，管理员也可以查看用户项目列表。

## 10. 等待体验与小游戏

- 生成过程中会展示实时进度。
- 生成完成后支持页面内提示、浏览器通知和后台标签页标题变化。
- 用户可以主动开启浏览器完成提醒；未授权通知时仍会显示页面内提示。
- 生成过程中用户可以玩小游戏，小游戏不消耗积分。
- 当前包含“AI 猜心”和“贪吃蛇”。
- AI 猜心支持多次猜测，猜不出时可以展示答案。
- 贪吃蛇结束后会给出一句点评。
- PPT 生成完成后不会直接清空小游戏区域，会提示用户 PPT 已生成，并提供跳转查看结果的按钮。

## 11. 账户、注册与登录

- 支持邮箱注册。
- 注册时需要输入两遍密码，两次密码不一致会阻止注册并提示。
- 支持邮箱 + 密码登录。
- 支持邮箱验证码登录。
- 发送验证码后有 60 秒冷却时间。
- 支持配置允许注册的邮箱后缀。
- 支持配置管理员邮箱列表。
- 登录态通过服务端 Session Cookie 维护。
- 数据库只保存密码摘要，不保存明文密码。
- 服务端使用随机 salt 和 PBKDF2 类密码哈希方式验证密码。

## 12. 免费额度与积分

- 未登录用户按 IP 记录每日免费额度。
- 未登录用户每天可免费生成 1 个 PPT。
- 未登录用户每天包含 1 次单页 AI 修改。
- 同一登录用户或匿名 IP 默认最多只能同时拥有 1 个排队中/生成中的 PPT 任务，避免重复提交把队列堵住。
- 注册用户默认获得新用户积分，当前设计值为 200。
- 登录用户每天可以签到一次，当前设计值为 30 积分。
- 签到后按钮会变成已签到状态。
- 一次文本规划、生成页或单页 AI 修改按 1 积分计。
- 一次 AI 生图按 5 积分计。
- PPT 生成启动前只校验余额，生成成功后才扣除 PPT 生成积分。
- PPT 生成失败或被管理员取消时不扣 PPT 生成积分；如果兼容历史任务中已经扣过，会按幂等退款记录退回。
- 积分变化通过账本记录，包含原因、引用对象和变更后余额。

## 13. 邀请制

- 注册用户拥有自己的邀请码。
- 用户可以查看邀请码、总邀请数、已奖励邀请数和待奖励邀请数。
- 被邀请用户注册后会建立邀请关系。
- 邀请奖励在被邀请用户完成第一次成功 PPT 生成后发放。
- 邀请人和被邀请人的奖励积分由后端配置控制。
- 注册奖励、每日签到、生成扣费、生图扣费和邀请奖励可以通过管理员积分规则调整。

## 14. 管理后台

- 项目保留单一 `admin` 角色，不再区分 `superadmin`。
- 管理员可以查看用户列表。
- 管理员可以按邮箱搜索用户。
- 管理员可以查看用户状态、角色、积分、项目数和生成次数。
- 管理员可以启用或禁用用户。
- 管理员可以把用户设为管理员或恢复为普通用户。
- 管理员可以手动调整用户积分并填写原因。
- 管理员可以查看用户积分流水。
- 管理员可以查看用户项目列表。
- 管理员可以查看排队、运行中、失败、取消和已完成的生成任务。
- 管理员任务面板会显示失败分类、失败摘要和任务耗时。
- 管理员可以手动取消排队中或运行中的生成任务，取消后按规则退还已扣积分。
- 管理员可以发布、归档和查看公告，已发布公告会显示在首页。
- 管理员可以查看全局 Dashboard 指标，包括用户数、项目数、任务数和积分余额池。
- 管理员可以配置积分规则，包括注册奖励、签到奖励、生成扣费、生图扣费和邀请奖励。
- 管理员可以配置底层 LLM / AI 生图 Provider，API Key 加密存储，前端只显示掩码。
- 管理员可以查看管理操作审计日志。

- 默认主题、默认需求说明、生成进度和空状态提示会随语言切换。
- 前端支持主题切换，当前主题包括：

  - 天空雾蓝 / Sky Blue
  - 薄荷清晨 / Lime

## 16. 后端与接口

后端基于 FastAPI，主要接口模块包括：

- `/api/health`：服务健康检查。
- `/api/health/db`：数据库健康检查。
- `/api/auth/*`：验证码、注册、登录、退出、当前用户。
- `/api/credits/*`：积分余额、积分流水、每日签到。
- `/api/invites/me`：我的邀请码和邀请统计。
- `/api/sessions/*`：项目创建、列表、详情、活动任务、删除。
- `/api/generation/*`：启动生成、查询状态、SSE 进度事件。
- `/api/assets/*`：素材上传、列表、更新和文件访问。
- `/api/slides/*`：单页 AI 修改、元素修改、插图、图片放置、历史版本和回滚。
- `/api/exports/*`：PPTX 导出、导出任务状态和下载。
- `/api/styles`：获取风格预设。
- `/api/announcements`：公开公告列表。
- `/api/admin/*`：管理员用户管理、积分调整、用户项目、生成任务、任务取消、公告、积分规则、Provider 配置和审计日志。

## 17. 数据与后台任务

- PostgreSQL 用作主数据库。
- Redis 用作 Celery Broker 和结果后端。
- Celery 负责 PPT 生成和 PPTX 导出等耗时任务。
- 生成任务和导出任务使用独立队列，避免 PPTX 导出堵住 PPT 生成。
- 生产环境生成 Worker 默认并发为 8，导出 Worker 默认并发为 1。
- Celery 已设置 `worker_prefetch_multiplier=1`，减少单个 Worker 预取过多任务导致排队不均。
- 上传文件、生成素材和导出文件存储在 `storage/`。
- 支持 Alembic 数据库迁移。

## 18. 部署与运维

- 生产域名为 `https://ppt.forgespark.org`。
- Nginx 负责静态前端和 `/api` 反向代理。
- 生产环境只建议公开 `80` 和 `443`。
- FastAPI 监听本机 `127.0.0.1:6001`。
- PostgreSQL 和 Redis 只监听本机端口，不直接暴露公网。
- 后端 API 和 Worker 通过 systemd 管理：
  - `tokenvizppt-api`
  - `tokenvizppt-worker`：PPT 生成队列。
  - `tokenvizppt-export-worker`：PPTX 导出队列。
- 常用重启命令：

```bash
sudo systemctl restart tokenvizppt-api tokenvizppt-worker tokenvizppt-export-worker
```

- HTTPS 使用免费证书，并配置自动续费。
- 前端更新流程是重新构建 `frontend/dist` 后 reload Nginx。
